# recover-ci-failure

## 用途

读取失败的 CI 摘要，判断是否可以在当前分支自动修复；若风险过高或信息不足，则显式升级人工。

## 输入

- 失败的 CI / checks 摘要
- 当前工作区
- 任务上下文

## 必要输出

- 恢复策略判断
- 可恢复时的补丁与验证摘要
- 不可恢复时的升级人工原因

## 约束

- 先判断自动恢复是否安全
- 不要盲目重试失败流水线
- 无法安全恢复时必须明确阻塞原因
