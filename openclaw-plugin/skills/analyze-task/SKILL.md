# analyze-task

## 用途

读取归一化任务载荷，识别受影响的文件或模块，暴露缺失信息，并生成可供 `implement-task` 执行的计划。

## 输入

- 归一化任务元数据
- 完整任务正文
- 仓库上下文

## 必要输出

- 结构化执行计划
- 受影响文件或模块
- 缺失信息列表
- 风险等级

## 约束

- 保持与仓库现有模式一致
- 不得跳过缺失信息报告
- 只有计划细化到可执行程度时，才可以声称实现已准备就绪
